## 一、内置对象
Helm 模板提供了一系列内置对象，用于动态生成 K8s 资源清单。以下是常用对象及其用法：
### 1. `.Release` 对象  
包含与当前 Helm **发布**（安装/升级操作）相关的信息。

**常用字段**：

- `.Release.Name`: 当前发布的名称（即 `helm install` 或 `helm upgrade` 时指定的名称）。
- `.Release.Namespace`: 发布的目标命名空间。
- `.Release.IsUpgrade`: 布尔值，表示当前操作是否为升级（`true`/`false`）。
- `.Release.IsInstall`: 布尔值，表示当前操作是否为安装（`true`/`false`）。
- `.Release.Revision`: 当前发布的修订版本号（安装时为 `1`，每次升级递增）。
- `.Release.Service`: 固定字符串 `"Helm"`，表示由 Helm 管理。

**示例**：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ .Release.Name }}-deployment  # 渲染为：xxx-deployment
  namespace: {{ .Release.Namespace }}  # 渲染为：kube-system（假设安装时指定了命名空间）
  labels:
    x: {{ .Release.IsUpgrade }}        # 首次安装时为 false
    y: {{ .Release.IsInstall }}        # 首次安装时为 true
    z: {{ .Release.Revision }}         # 首次安装时为 1
    a: {{ .Release.Service }}          # 固定为 "Helm"
```

**使用场景**：

- 动态生成资源名称（如 `name: {{ .Release.Name }}-app`）。
- 根据操作类型（安装/升级）调整配置。
### 2. `.Values` 对象  
包含用户传入的配置值，支持多种来源（优先级从高到低）：
1. `--set` 命令行参数。
2. 自定义 `values.yaml` 文件。
3. Chart 包内默认的 `values.yaml`。

**示例**：

```yaml
# values.yaml
replicaCount: 2
image:
  repository: nginx
  tag: stable
```

```yaml
# templates/deployment.yaml
spec:
  replicas: {{ .Values.replicaCount }}
  containers:
  - name: web
    image: {{ .Values.image.repository }}:{{ .Values.image.tag }}
```

**使用方式**：

- **命令行传值**：  
  ```bash
  helm install myapp --set replicaCount=3 .
  ```
- **自定义文件传值**：  
  ```bash
  helm install myapp -f custom-values.yaml .
  ```
### 3. `.Chart` 对象  
包含 Chart 自身的元数据信息，从 `Chart.yaml` 中读取。

###### 常用字段：
- `.Chart.Name`: Chart 名称（`Chart.yaml` 中的 `name`）。
- `.Chart.Version`: Chart 版本（`Chart.yaml` 中的 `version`）。
- `.Chart.AppVersion`: 应用版本（`Chart.yaml` 中的 `appVersion`）。

**示例**：

```yaml
labels:
  chart: {{ .Chart.Name }}-{{ .Chart.Version }}  # 输出：mychart-0.1.0
  appVersion: {{ .Chart.AppVersion }}            # 输出：1.16.0（假设应用版本为 1.16.0）
```
### 4. `.Files` 对象  
用于访问 Chart 包内的文件内容，支持文本文件的读取和简单处理。

**常用方法：**

- `.Files.Get "path/file.txt"`: 读取文件内容。
- `.Files.GetBytes`: 以字节数组形式读取文件（适用于二进制文件）。
- `.Files.Glob`: 使用通配符匹配文件列表。
- `.Files.Lines`：逐行读取文件内容（适用于行遍历）

**示例**：

```yaml
containers:
- name: app
  env:
  - name: USER
    value: {{ .Files.Get "config/username.txt" | trim }}  # 读取并去除空格
  - name: PASS
    value: {{ .Files.Get "config/password.txt" | b64enc }} # 读取并 Base64 编码
```

**注意**：

- 文件需存放在 Chart 目录中（如 `mychart/config/username.txt`）。
- 默认不能读取 `templates/` 和 `.helmignore` 排除的文件。
### 5. `.Capabilities` 对象  
提供 K8s 集群的版本和功能支持信息。

###### 常用字段：
- `.Capabilities.KubeVersion`: 集群版本信息（如主版本号 `Major: "1"`, 次版本号 `Minor: "30"`, `GitVersion: "v1.30.0"`）。
- `.Capabilities.APIVersions`: 集群支持的 API 版本列表。
- `.Capabilities.APIVersions.Has $version`：判断一个版本（如 `batch/v1`）或资源（比如 `apps/v1/Deployment`）是否可用。

  - > `apps/v1` (Deployment, StatefulSet) 用于管理长期运行的服务，目标是保持指定数量的副本持续运行。`batch/v1` 用于管理短期任务，目标是任务成功完成并退出。


###### 示例：
```yaml
apiVersion: v1
kind: Service
metadata:
  labels:
    kubeVersion: {{ .Capabilities.KubeVersion.GitVersion }}  # 输出：v1.30.0
    majorVersion: {{ .Capabilities.KubeVersion.Major }}       # 输出：1
    minorVersion: {{ .Capabilities.KubeVersion.Minor }}       # 输出：30
    hasBatchV1: {{ .Capabilities.APIVersions.Has "batch/v1" }} # 检查是否支持 batch/v1 API
```

###### 使用场景：
- 根据集群版本动态调整 API 版本（如 `apps/v1` 与 `extensions/v1beta1`）。
- 条件渲染资源（如仅当集群支持某个 API 时创建资源）。
### 6. `.Template` 对象  
包含当前模板文件的路径信息，用于调试或动态生成内容。

**常用字段：**

- `.Template.Name`: 当前模板文件的完整路径（相对于 Chart 根目录）。
- `.Template.BasePath`: 当前模板所在的目录路径。

**示例：**

```yaml
metadata:
  labels:
    templatePath: {{ .Template.Name }}   # 输出：mychart/templates/deployment.yaml
    basePath: {{ .Template.BasePath }}   # 输出：mychart/templates
```
## 二、函数与管道

Helm 模板支持通过**函数**和**管道**对数据进行动态处理，增强模板的灵活性。
### 1. **函数调用**
- **语法**：`{{ 函数名 参数1 参数2 ... }}`  
  
  直接调用函数，参数可以是变量或静态值。
  
- **示例**：
  
  ```yaml
  # 调用内置函数 `template`，引用其他模板，不支持管道
  {{ template "mychart.name" . }}  
  # 引用并渲染名为 mychart.name 的模板，并将当前上下文（.）传递给该模板
  # 该模板来自templates/_helpers.tpl，内含多个辅助模板（类似一段代码逻辑/函数），它们可以在chart的其他部分重用
  # 而 mychart.name 用于生成资源的名称，使用 .Values.nameOverride 如果设置了，否则使用 .Chart.Name。还对名称进行了截断和转换为小写。（其逻辑）
  ```
### 2. **管道操作**
- **语法**：`{{ 值 | 函数名 | 函数名 ... }}`  
  通过管道符 `|` 将前一个函数的输出作为下一个函数的输入。
- **示例**：
  
  ```yaml
  # 链式处理：转大写 -> 重复 3 次 -> 加引号
  food: {{ .Values.favorite.food | upper | repeat 3 | quote }}
  
  # quote：为值添加双引号，防止 YAML 解析错误。
  # upper：将字符串转为大写。
  # repeat：重复字符串指定次数。
  ```
  - **输入**：若 `.Values.favorite.food = "pizza"`  
  - **输出**：`food: "PIZZAPIZZAPIZZA"`
### 3. 综合示例

```yaml
# values.yaml
favorite:
  # food: "pizza"  # 注释此行以触发默认值

# templates/configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-config
data:
  food: {{ .Values.favorite.food | default (printf "%s-xxx" (include "mychart.name" .)) | upper | quote }}
  kubeVersion: {{ .Capabilities.KubeVersion.Major | quote }}
  
# default：设置默认值，当变量未定义时使用。（若变量已经被定义了，则会忽略该函数）
# printf：格式化字符串，支持动态拼接。语法：printf "格式化规则" 参数1 (参数2)...
# include：包含其他模板文件的内容。（与 template 作用相同，但是可以支持管道）
```

- **渲染结果**（假设 `include "mychart.name"` 返回 `myapp`）：

  ```yaml
  apiVersion: v1
  kind: ConfigMap
  metadata:
    name: myrelease-config
  data:
    food: "MYAPP-XXX"  # 使用动态默认值
    kubeVersion: "1"    # Kubernetes 主版本
  ```
### 4. 注意事项

1. **不支持管道的函数**：  

   部分函数（如 `template`）因需传递上下文（`.`）无法使用管道，需直接调用。

   ```yaml
   {{ template "mychart.name" . }}  # 正确
   {{ . | template "mychart.name" }} # 错误
   ```

2. **参数顺序**：  

   管道操作的顺序影响结果，需按逻辑链排列函数。

   ```yaml
   # 错误示例：先加引号后转大写会导致引号内内容不处理
   food: {{ .Values.food | quote | upper }}  # 输出：`"CAKE"`（引号也被转大写）
   ```

3. **动态默认值**：  

   `default` 的参数可以是复杂表达式，需用 `()` 包裹。

   ```yaml
   # 正确：使用括号明确参数范围
   {{ .Values.food | default (include "name" .) }}
   ```

4. **YAML 兼容性**：  

   使用 `quote` 处理含特殊字符（如 `:`、`{}`）的字符串，避免解析错误。
## 三、流程控制
### 1.  if/else
#### 1.1 基本语法

```yaml
{{- if 条件1 }}
  # 条件1 为 true 时执行
{{- else if 条件2 }}
  # 条件2 为 true 时执行
{{- else }}
  # 所有条件为 false 时执行
{{- end }}
```
- **注意**：`{{-` 和 `-}}` 用于控制空白符，确保生成的 YAML 格式正确。
#### 1.2 条件类型
Helm 支持多种条件判断方式，包括逻辑运算、比较运算、函数组合等。
##### （1）**布尔值判断**
直接使用布尔值或返回布尔值的表达式。
```yaml
{{- if .Values.db.debugMode | not }}
  name: DB_DEBUG1
  value: "false"
{{- else }}
  name: DB_DEBUG2
  value: "true"
{{- end }}
```
- **解析**：
  - `.Values.db.debugMode` 为 `false` 时，`not` 返回 `true`，渲染 `DB_DEBUG1`。
  - `.Values.db.debugMode` 为 `true` 时，渲染 `DB_DEBUG2`。
##### （2）**比较运算**
使用比较函数判断值之间的关系：
- `eq`: 等于
- `ne`: 不等于
- `lt`: 小于
- `gt`: 大于
- `le`: 小于等于
- `ge`: 大于等于

###### 示例：
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  {{- if eq .Values.favorite.drink "coffee" }}
  mug: true
  {{- end }}
```
- **解析**：若 `favorite.drink` 值为 `"coffee"`，则渲染 `mug: true`。
##### （3）**逻辑运算**
通过 `and`、`or` 组合多个条件，支持使用 `()` 分组逻辑。
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  {{- if and (eq .Values.favorite.drink "coffee") (gt (int .Values.replicaCount) 3) }}
  mug: true
  {{- end }}
```
- **解析**：
  - `and` 要求所有条件为 `true`。
  - `eq .Values.favorite.drink "coffee"`: 检查 `drink` 是否为 `"coffee"`。
  - `gt (int .Values.replicaCount) 3`: 将 `replicaCount` 转为整数后检查是否大于 3。
#### 1.3 完整示例
##### **场景**：根据条件动态生成 ConfigMap
```yaml
# values.yaml
favorite:
  drink: "coffee"
replicaCount: "5"  # 字符串类型
debugMode: false
```

```yaml
# templates/configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  {{- if and (eq .Values.favorite.drink "coffee") (gt (int .Values.replicaCount) 3) }}
  mug: "true"
  {{- else }}
  mug: "false"
  {{- end }}
  {{- if not .Values.debugMode }}
  logLevel: "info"
  {{- end }}
```

##### **渲染结果**：
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: myrelease-configmap
data:
  mug: "true"    # 条件1：drink=coffee 且 replicaCount>3
  logLevel: info  # 条件2：debugMode=false
```
### 2. 空格控制
Helm 模板通过 `{{-` 和 `-}}` 符号控制渲染后的空白符，确保生成的 YAML 格式正确，避免解析错误。
#### 2.1 **空白符删除**

- **`{{-`**: 删除左侧的空白和换行符。

- **`-}}`**: 删除右侧的空白和换行符。

- **示例**：

  ```yaml
  data:
    {{- if eq .Values.drink "coffee" }}  # 删除左侧换行和缩进
    mug: true
    {{- end }}                       
  ```

  渲染后：

  ```yaml
  data:
    mug: true
  # 否则：
  data:
  
    mug: true
    
  ```
#### 2.2 **缩进控制函数**

- **`indent N`**: 将文本向右缩进 `N` 个空格。

- **`nindent N`**: 插入换行符后缩进 `N` 个空格。

- **示例**：

  ```yaml
  {{- if eq .Values.drink "coffee" }}
  {{ nindent 2 "mug: true" }}  # 插入换行并缩进 2 空格
  {{- end }}
  ```

  渲染后：

  ```yaml
    mug: true
  ```
### 3. 使用 with 修改作用域
`with` 用于限定变量作用域，简化嵌套路径的访问。

```yaml
# values.yaml
db:
  debugMode: true
  dbinfo:
    user: xxx
    pass: "123"
    port: 3306

# templates/configmap.yaml
data:
  {{- with .Values.db.dbinfo }}  # 作用域限定为 .Values.db.dbinfo
  user: {{ .user }}              # 直接访问 .user
  pass: {{ .pass }}
  port: {{ .port }}
  {{- end }}
  release: {{ .Release.Name }}   # 外部变量需在 with 外访问
```

渲染后：

```yaml
data:
  user: xxx
  pass: "123"
  port: 3306
  release: myrelease
```
### 4. range 循环
`range` 用于遍历列表或字典，支持多种迭代模式。
#### 4.1 **遍历列表**

##### **示例 1：遍历列表元素**

```yaml
# values.yaml
names:
  - lili
  - jack

# 模板
nameList1: |-
  {{- $relname := .Release.Name -}} # “.”:全局变量
  {{- range .Values.names }}
  - "{{ . | title }}-{{ $relname }}"  # . 表示当前元素
  {{- end }}
```

渲染后：

```yaml
nameList1: |-
  - "Lili-myrelease"
  - "Jack-myrelease"
```

##### **示例 2：遍历固定元组**

```yaml
nameList2: |-
  - {{ . }}-{{ $.Release.Name }}  # 使用 $ 访问全局对象
  {{- end }}
```

渲染后：

```yaml
nameList2: |-
```
#### 4.2 **遍历字典**

##### **示例 3：遍历值**

```yaml
# values.yaml
favorite:
  drink: coffee
  food: pizza

# 模板
nameList3: |-
  {{- range .Values.favorite }}
  - {{ . | quote }}  # . 表示值
  {{- end }}
```

渲染后：

```yaml
nameList3: |-
  - "coffee"
  - "pizza"
```

##### **示例 4：遍历键值对**

```yaml
nameList4: |-
  {{- range $key, $value := .Values.favorite }}
  {{ $key }}: {{ $value | quote }}
  {{- end }}
```

渲染后：

```yaml
nameList4: |-
  drink: "coffee"
  food: "pizza"
```

##### **示例 5：遍历带索引的列表**

```yaml
nameList5: |-
  {{- range $index, $value := .Values.names }}
  {{ $index }}: {{ $value | quote }}
  {{- end }}
```

渲染后：

```yaml
nameList5: |-
  1: "lili"
  2: "jack"
```
#### 4.3 **关键细节**

- **作用域变化**：在 `range` 循环内，`.` 指向当前元素，需通过 `$` 访问全局对象。
- **变量保存**：在循环外定义变量（如 `$relname`），避免重复计算。
- **多行字符串**：使用 `|-` 保留换行但去除末尾空行，`|` 保留所有换行。

## 四、变量
### 1. 全局作用域
在 Helm 模板中，`$` 始终指向**根上下文**（即全局作用域），即使进入 `with` 或 `range` 等作用域块后，仍可通过 `$` 访问全局对象。
#### 1.1 **`with` 块中的全局访问**
当进入 `with` 块后，`.` 指向局部作用域，需用 `$` 访问全局对象。
```yaml
# values.yaml
db:
  dbinfo:
    pass: "123"

# 模板示例
data:
  {{- with .Values.db.dbinfo }}
  user: {{ .user }}           # 正确：局部作用域下的 .user
  pass: {{ .pass }}
  release: {{ $.Release.Name }}  # 正确：通过 $ 访问全局 Release 对象
  {{- end }}
```
- **渲染结果**：
  ```yaml
  data:
    pass: "123"
    release: myrelease  # 全局变量值
  ```
#### 1.2 **`range` 循环中的全局访问**
在 `range` 循环中，`.` 指向当前迭代项，需用 `$` 访问全局对象。
```yaml
# values.yaml

# 模板示例
nameList1: |-
  {{- range .Values.names }}
  - "{{ . | title }}-{{ $.Release.Name }}"  # . 为当前元素，$ 访问全局
  {{- end }}
```
- **渲染结果**：
  ```yaml
  nameList1: |-
    - "Lili-myrelease"
    - "Jack-myrelease"
  ```
### 2. 变量定义与作用域
Helm 支持通过 `$variable := value` 定义变量，变量作用域限定在定义它的块内。
#### 2.1 **显式变量定义**
```yaml
# 模板示例
data:
  {{- $relName := .Release.Name -}}  # 定义全局变量
  release: {{ $relName }}            # 直接使用
  {{- range .Values.names }}
  - {{ . | title }}-{{ $relName }}   # 在循环中使用预存变量
  {{- end }}
```

#### 2.2 **隐式变量（`range` 中的键值对）**
在 `range` 循环中可直接定义键、值、索引变量：
```yaml
# 遍历字典（键值对）
nameList4: |-
  {{- range $key, $value := .Values.favorite }}
  {{ $key }}: {{ $value | quote }}  # 显式使用键值变量
  {{- end }}

# 遍历列表（带索引）
nameList5: |-
  {{- range $index, $value := .Values.names }}
  {{ $index }}: {{ $value | quote }}  # 显式使用索引和值变量
  {{- end }}
```
- **渲染结果**：
  
  ```yaml
  nameList4: |-
    drink: "coffee"
    food: "pizza"
  nameList5: |-
    1: "lili"
    2: "jack"
  ```
## 五、全局模板与复用
### 1. 模板定义与复用
Helm 允许通过 `_helpers.tpl` 文件定义**全局可复用的模板片段**，提升代码复用性和可维护性。该文件可以**被其他 chart 包调用**，也可以**为本 chart 下templates 内的其他清单文件提供全局的命名**。
#### 1.1 **模板定义语法**
在 `templates/_helpers.tpl` 文件中定义模板：
```yaml
{{- define "模板名称" }}
  # 模板内容
{{- end }}
```
- **示例 1**：简单模板（无需上下文）  
  
  ```yaml
  {{- define "mychart.labels" }}
    labels:
      generator: helm
      date: {{ now | htmlDate }}  # 使用内置函数 now 和 htmlDate 格式化日期
  {{- end }}
  ```
  
- **示例 2**：依赖上下文/作用域的模板  
  
  ```yaml
  {{- define "mychart.labels" }}
    labels:
      generator: helm
      date: {{ now | htmlDate }}
      chart: {{ .Chart.Name }}      # 需要传入上下文以访问 .Chart
      version: {{ .Chart.Version }}
  {{- end }}
  ```
#### 1.2 **模板调用方式**
##### **`template` 函数**
- **语法**：`{{ template "模板名称" 上下文 }}`  
- **特点**：直接插入模板内容，不支持管道操作。
- **示例**：
  
  ```yaml
  metadata:
    name: {{ .Release.Name }}-configmap
    {{- template "mychart.labels" . }}  # 传入当前上下文（.）
  ```
  - **渲染结果**：
    ```yaml
    metadata:
      name: myrelease-configmap
      labels:
        generator: helm
        date: 2023-10-05
        chart: mychart
        version: 0.1.0
    ```

##### **`include` 函数**
- **语法**：`{{ include "模板名称" 上下文 | 处理函数 }}`  
- **特点**：支持管道操作，可对模板输出进行二次处理（如缩进、拼接）。
- **示例**：
  ```yaml
  metadata:
    name: {{ .Release.Name }}-configmap
    labels:
      {{- include "mychart.labels" . | indent 2 }}  # 缩进 2 个空格
      
  # 若yaml文件解析失败，可以通过 `helm template --debug` 检查渲染后的格式。
  ```
### 2. 完整示例
##### **`_helpers.tpl` 文件**
```yaml
{{- define "mychart.labels" }}
  labels:
    generator: helm
    date: {{ now | htmlDate }}
    chart: {{ .Chart.Name }}
    version: {{ .Chart.Version }}
{{- end }}
```

##### **`cm.yaml` 模板文件**
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
  {{- include "mychart.labels" . }}
data:
  myvalue: "Hello World"
```

##### **渲染结果**
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: myrelease-configmap
  labels:
    generator: helm
    date: 2023-10-05
    chart: mychart
    version: 0.1.0
data:
  myvalue: "Hello World"
```
## 六、访问/引入外部普通文件
### 1. 注意事项
Helm 通过 `.Files` 对象访问 Chart 目录中的普通文件，但需遵循以下规则：
1. **文件大小限制**  
   
   - K8s 限制单个对象大小，Chart 包（含文件）需小于 **1M**。
   
   - **常见陷阱**：避免在 Chart 目录中存放镜像文件等大文件，否则 `helm install` 可能报错：  
     
     `hcreate: failed to create: Request entity too large: limit is 3145728`
   
2. **禁止访问的路径**  
   - `templates/` 目录下的文件。
   - `.helmignore` 文件中排除的文件。
### 2. 准备工作
创建示例文件与目录结构：
```bash
# 进入 Chart 目录
cd mychart/

# 创建外部配置文件
cat > config1.toml << EOF
message = Hello from config 1
EOF

cat > config2.toml << EOF
message = This is config 2
EOF

cat > config3.toml << EOF
message = Goodbye from config 3
EOF

# 创建嵌套文件
mkdir foo
cat > foo/bar.txt << EOF
1111
2222
3333
EOF
```

目录结构：
```
mychart/
├── charts
├── Chart.yaml
├── config1.toml
├── config2.toml
├── config3.toml
├── templates
│   └── cm.yaml
├── values.yaml
└── foo
    └── bar.txt
```
### 3. 使用示例
#### 3.1 **直接引入文件内容**
将多个文件内容注入 ConfigMap：
```yaml
# templates/cm.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  config1.toml: |-
    {{ .Files.Get "config1.toml" | indent 4 }}
  config2.toml: |-
    {{ .Files.Get "config2.toml" | indent 4 }}
  config3.toml: |-
    {{ .Files.Get "config3.toml" | indent 4 }}
```
#### 3.2 **循环批量引入文件**
使用 `tuple` 遍历文件名列表：
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  {{- range tuple "config1.toml" "config2.toml" "config3.toml" }}
  {{ . }}: |-
    {{ $.Files.Get . | indent 4 }}  # 通过 $ 访问全局 Files 对象
  {{- end }}
```
- **渲染结果**：每个文件内容作为独立键值对，自动缩进 4 空格。
#### 3.3 **Base64 编码文件内容**
对文件内容进行加密处理：
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  {{- range tuple "config1.toml" "config2.toml" "config3.toml" }}
  {{ . }}: |-
    {{ $.Files.Get . | b64enc | indent 4 }}  # 使用 b64enc 编码
  {{- end }}
```
#### 3.4 **逐行处理文件内容**
使用 `.Files.Lines` 按行读取文件：
```yaml
# 逐行处理
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ .Release.Name }}-configmap
data:
  some-file.txt: |
    {{- range .Files.Lines "foo/bar.txt" }}
    {{ . }}  # 每行单独处理，自动缩进
    {{- end }}
```
- **渲染结果**：
  ```yaml
  data:
    some-file.txt: |
      1111
      2222
      3333
  ```
## 七、`NOTES.txt` 文件使用
### 1. 作用与功能
`NOTES.txt` 是 Helm Chart 的**特殊模板文件**，用于在 `helm install` 或 `helm upgrade` 完成后向用户显示提示信息，例如：
- 应用访问方式
- 后续操作指南
- 调试命令
### 2. 文件位置与命名
- **路径**：必须位于 `templates/` 目录下。
- **文件名**：必须全大写 `NOTES.txt`（区分大小写）。
### 3. 模板语法
支持所有 Helm 模板语法，可访问内置对象（如 `.Chart`、`.Release`）。
##### 示例模板
```text
# templates/NOTES.txt
Thank you for installing {{ .Chart.Name }}.

Your release is named {{ .Release.Name }}.

To learn more about the release, try:
  $ helm status {{ .Release.Name }}
  $ helm get {{ .Release.Name }}
```

##### 安装后输出
```bash
$ helm install myrelease .
NAME: myrelease
LAST DEPLOYED: Sat Aug 31 11:56:53 2024
NAMESPACE: default
STATUS: deployed
REVISION: 1
NOTES:
Thank you for installing mychart.  # .Chart.Name 替换为 Chart 名称

Your release is named myrelease.   # .Release.Name 替换为发布名称

To learn more about the release, try:
  $ helm status myrelease
  $ helm get myrelease
```

##### **调试与测试**
- **预览输出**：
  ```bash
  helm install --dry-run myrelease .  # 查看渲染后的 NOTES 内容
  ```

- **查看已部署的 NOTES**：
  ```bash
  helm get notes myrelease  # 显示指定 Release 的 NOTES
  ```

- **模板错误处理**：

  若模板语法错误，`helm install` 会报错，需通过 `helm lint` 检查。
